WORKFLOW CLARIFICATION:
=======================

PAYMENT TRANSACTIONS TAB:
-------------------------
For regular payments (GCash, Cash, etc.):
- Button: ✓ (CheckCircle icon)
- Tooltip: "Verify payment"
- Result: Status changes to "Paid"

For PDO payments (pending approval):
- Button: ✓ (CheckCircle icon - DISABLED)
- Tooltip: "PDO must be approved first in PDO Management tab"
- Must go to PDO Management first

For PDO payments (awaiting payment):
- Button: ✓ (CheckCircle icon - ENABLED)
- Tooltip: "Mark PDO check as paid/cleared"
- Result: Status changes to "Paid"


PDO MANAGEMENT TAB:
-------------------
STAGE 1 - Pending Approval:
- Check submitted by customer
- Action Buttons:
  * ✓ (CheckCircle) - "Approve Check"
  * ✕ (XCircle) - "Reject Check"
- Result: Moves to "Awaiting Payment" section

STAGE 2 - Awaiting Payment:
- Check approved, waiting to clear
- Action Button:
  * ✓ (CheckCircle) - "Mark as Paid"
- Result: Moves to "Verified" section with "Paid" status

STAGE 3 - Verified (Completed):
- Check cleared and paid
- Status Badge: "Paid"
- No action buttons (read-only)


CONSISTENT TERMINOLOGY:
=======================

OLD (Confusing):
- Action: "Verify" → Status: "Verified"
- Action: "Mark as Paid" → Status: "Verified" ❌ INCONSISTENT

NEW (Clear):
- Action: "Verify payment" → Status: "Paid" ✅
- Action: "Mark as Paid" → Status: "Paid" ✅

EXAMPLE USER FLOW:
==================

Scenario 1: GCash Payment
--------------------------
1. Customer pays ₱1,000 via GCash
2. Uploads proof of payment
3. Payment appears in "Payment Transactions" with status "Pending Verification"
4. Admin clicks ✓ button (tooltip: "Verify payment")
5. Status changes to "Paid" ✅
6. Order balance updated

Scenario 2: PDO Payment (Full)
-------------------------------
1. Customer pays ₱5,000 with PDO check
2. Check appears in "PDO Management → Pending Approval"
3. Admin clicks ✓ button (tooltip: "Approve Check")
4. Check moves to "PDO Management → Awaiting Payment"
5. Days later, check clears at bank
6. Admin clicks ✓ button (tooltip: "Mark as Paid")
7. Check moves to "PDO Management → Verified" with status "Paid" ✅
8. Order balance updated

Scenario 3: PDO Payment (Installment)
--------------------------------------
1. Customer has 5 installment plan
2. Pays Installment #1 (₱1,000) with PDO check
3. Shows in PDO Management as "Installment: #1"
4. Admin approves check
5. Moves to "Awaiting Payment"
6. Check clears, admin marks as paid
7. Status: "Paid" ✅
8. Payment Plans updates: "0/5 paid" → "1/5 paid"
9. Repeat for remaining installments
